Models for Classical Computation

To start off, when computer scientists say computing, they really meant some form of machines (computer) that can solve problem. Some of which are:

  • Turing Machine
  • Finite Automata
  • $\lambda$-calulus
  • Coin
  • Soap Bubbles and Pegs (my favourite, read the paper by Scott Aaronson for more info)
  •  $\vdots$

 In 1936 Turing came up with the famous Turing Universal Computation. At that time, Turing was interested in solving some obscure problems related with the decision problems and he noticed that some model of computation are universal, in other word, they can sufficiently simulate other models of computation. However, there is one more model which was mainly used for Quantum Computing that I didn't mention, the curcuit model. Actually, the cucuit model was consider as "super universal", meaning not only it can sufficiently simulate Turing Machine, it could also do computation that would be considered to be impossible for a Turing Machine.

Overview of Turing Machine, consisting of an infinite tape, a head that can be made of finite automata and the ability to read and write on the tape.

 So, what is the circuit model? Well, all freshmen in Computer Science are taught in their first class of Discrete Mathematics that the fundamental boolean operator can be using a combination of only two gates: AND and OR gate (functional completeness of logical operators). Moreover, these logic gates has been used by electrical engineers well before Turing's work. So, if this model is so useful, why didn't Turing used it in his original paper? According to Peter Shor, since circuits are only designed for finite size inputs, computation that is independent of the input size (like which the Turing Machine was designed for) is impossible by definition. However, this unique "bug" of the curcuits model can also be seen as a feature to solve problems that are considered as "incomputable" by Turing Machine. Consider the infamous Halting Problem:

$\textrm{Does a Turing Machine with a program given as } P \textrm{ and input } i \textrm{ halt? }$

 In other words, given two program, A and B, can we tell that if A(B) halts before the universe collapse. We can show this is incomputable by proof of contradiction:

Assuming the Halting Problem is computable, there exist an Turing Machine $M$ which solve the problem. Now we can tell that any input $i$ halts or not. We can make another program $M_{1}$ that have the following properties:

  • run forever if $M(i)$ halts for some input $i$
  • halts if $M(i)$ run forever for some input $i$

Then we find that $M(M_{1}(M_{1}))$ fails to tell us whether $M_{1}(M_{1})$ halts or not. Thus, we arrive at a contradiction.$\tag*{$\blacksquare$}$

 This can be solved with a family of circuits: circuits with $2^{P\cdot i}$ input and output $1$ if the TM halts and 0 if it doesn't. The trick is: for every set of input and output, there is a different family of circuits, and it can hide the incomputability information within those circuits.

 Actually, in 1994 when Peter Shor came up with his algorithm, he used a quantum turing machine in which the head is in coherent superposition as it is moving along the tape. It was this that many Phyisicts would reject the idea due to its insanely difficult engineerign design. Therefore, the circuits model was mostly used for Quantum Computing since we can have a few qubit in coherent superpostion going through different gates. Well you might still argue that this is still impossible to build even with today's technology but back then it was $log(\textrm{impossible})$ compare to the QTM.

 Note that: the description of the circuit model should be an output of a classical program. This is intuitive cause you don't want a weird circuits that has some hidden uncomputable information in its description.

History of quantum mechanics and quantum computation

 The foundation of Quantum Mechanics was mainly developed around 1900 and 1930. In 1936, Einstein, Podolski, and Rosen published a paper about Quantum Mechanics is incomplete and that it's an approximation of a bigger theory. They came up with the Local Hidden-Variable argument which states that subatomic particles has some hidden-variable that allow them to exhibit the probabilistic behavior in previous theories. This arguement was, however, proven impossible in 1964 by John S. Bell with his famous Bell's Theorem. With experiment around 1980 performed by Aspect proving Bell correct. In 1982, Herbert published a paper called FLASH, showing the characteristic of entangled particles (ERP pair). Within the same year, No-Cloning Theorem was made to explain the flaw in this paper. The theorem said that an unknown quantum state cannot be duplicated. This is a result from the fact that we can only obtain partial information of the quantum system.
