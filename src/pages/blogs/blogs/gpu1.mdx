## GPU NoC (Network-on-Chip)

GPU consits of `SM` which form `TPC` which form `GPC`. With L1 on the SM
side after the crossbar. Before the crossbar are the memory units (L2, Memory
Controller). Overall computing components are hierarchical (TPC, GPC). Why?
Although the flat architecture is similar to multicore CPU, GPU are hierarchical
because we want to limit the interconnection network. Since threads are mostly
independent, communication between the memory and the SM can be reduced by introducing
block. The first thing in kindergarten is sharing! :) So although the SM are clustered,
to share between the SM, the bandwidth is shared among the SM. Mordern GPU mediate
this by increasing the bandwidth between the MUX and the SIMT and the memory.
Data is cache in L2. The SM can know its corresponding partitioned L2 by computing
the offset from memory address with a very complicated hashing function. We partition
memory as well as having this hashing function to make the memory access balanced
among the partition and thus increase the bandwidth.
